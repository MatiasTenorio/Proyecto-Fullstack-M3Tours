## Commits

Conventional Commits:

tipo(alcance): mensaje corto y claro

1. El Tipo (tipo):
    Indica qué clase de trabajo acaban de hacer.

- feat (Feature): Añadieron una nueva funcionalidad o característica.
- fix: Arreglaron un error o bug.
- chore: Tareas de mantenimiento (ej. actualizar dependencias, configurar el proyecto) que no cambian el código de la aplicación para el usuario.
- docs: Cambios exclusivos en la documentación (como actualizar el archivo README.md).
- refactor: Modificaron el código para mejorarlo internamente, pero sin añadir funcionalidades nuevas ni arreglar errores.

2. El Alcance (alcance):
    Indica qué microservicio o módulo exacto se acaba de modificar.

3. El Mensaje:
    Debe ser breve (idealmente de 50 caracteres o menos) y usar verbos en modo imperativo, como si le estuvieran dando una orden al código. Usen "agregar", "corregir", "eliminar", en lugar de "agregué", "corrigiendo" o "agregado".

## Instrucciones de Trabajo

** IMPORTANTE **
Asegurarse de trabajar en la rama correspondiente:
Si se va a desarrollar o cambiar algo dentro del microservicio "empresas", hacer el "git switch (rama correspondiente)".
Hacer los commits desde respectiva rama.
